# K-Racing 디자인 문서

## 개요
탑뷰 싱글 레이싱 게임. 고정된 타원형 트랙을 3바퀴 완주하는 시간을 측정하여 랭킹을 매기는 타임어택 방식.

## GAME_META
```typescript
export const GAME_META: TGameMeta = {
  id: 'kracing',
  title: 'K-Racing',
  engine: 'canvas',
  platform: 'both',
  touchControls: 'tap',
  orientation: 'portrait',
  category: 'arcade',
  difficulty: 'fixed',
};
```

## 게임 흐름
1. **시작 화면** → S키 / 화면 터치
2. **3초 카운트다운** (3... 2... 1... GO!)
3. **레이싱** → 3바퀴 완주까지 시간 측정
4. **결과 화면** → 총 시간 + 랩별 타임 + SAVE/SKIP

## 조작

| 플랫폼 | 입력 | 동작 |
|--------|------|------|
| 데스크탑 | ← → | 좌/우 회전 |
| 데스크탑 | Space | 브레이크 |
| 데스크탑 | S | 시작/재개 |
| 데스크탑 | P | 일시정지 |
| 데스크탑 | R | 재시작 |
| 모바일 | 좌측 터치 | 좌회전 |
| 모바일 | 우측 터치 | 우회전 |
| 모바일 | 양쪽 동시 터치 | 브레이크 |

## 물리 모델

### 자동 전진
- 차량은 항상 전방으로 가속 (최대 속도까지)
- 가속도와 최대 속도는 config에서 조절

### 회전
- 좌/우 키로 차량 각도(heading) 변경
- 회전 속도는 현재 속도에 비례 (정지 시 회전 불가)

### 브레이크
- Space / 양쪽 동시 터치로 급감속

### 드리프트
- 브레이크 + 회전 동시 입력 시 드리프트 발동
- 드리프트 중: 회전량 1.5배
- 드리프트 종료 후: +10% 속도 부스트
- 타이어 자국 이펙트 표시 (잔상 배열)

### 벽 충돌
- 트랙 벽에 부딪히면 속도 70% 감소
- 벽 안쪽으로 밀려남 (관통 불가)
- 시간 페널티 없음

## 트랙 설계

### 형태
- 중앙 타원형 오바트랙
- 내벽/외벽으로 구성된 폐회로
- 웨이포인트 기반으로 트랙 중심선 정의
- 트랙 폭(TRACK_WIDTH)으로 내벽/외벽 생성

### 충돌 감지
- 차량 위치에서 가장 가까운 트랙 중심선 점을 찾음
- 중심선까지의 거리가 TRACK_WIDTH/2를 초과하면 벽 충돌
- 충돌 시 차량을 트랙 안쪽으로 밀어냄

### 랩 카운트
- 시작/결승선 통과 감지 (트랙 중심선의 특정 세그먼트)
- 진행 방향으로만 카운트 (역주행 감지)

## 카메라 시스템

### 차량 중심 카메라
- 차량이 항상 캔버스 중앙에 위치
- 카메라 오프셋 = 차량 위치 - 캔버스 중앙
- ctx.translate()로 월드 좌표를 스크린 좌표로 변환

### 미니맵
- 좌상단에 작은 크기로 전체 트랙 표시
- 트랙을 축소하여 그리고 현재 위치를 점으로 표시

## HUD 구성

### 인게임 HUD
- 상단 중앙: 총 경과 시간 (00:00.000)
- 상단 우측: LAP 카운터 (1/3)
- 좌상단: 미니맵 (트랙 전체 + 현재 위치)
- 하단 우측: 속도 게이지 바

### 카운트다운
- 3... 2... 1... GO! 순서로 화면 중앙에 큰 텍스트 표시
- 각 숫자 1초씩, GO!는 0.5초 후 사라짐

### 결과 화면
- FINISH! 텍스트
- Total: 00:45.320
- Lap 1~3 개별 타임
- SAVE/SKIP 버튼 (기존 createGameOverHud 패턴)

## 점수 체계

- 밀리초 단위 정수로 저장 (45.320초 → 45320)
- 랭킹 보드: ASC 정렬 (낮을수록 상위)
- 표시 형식: "45.320s"

## 비주얼

### 색상 팔레트
- 배경: #1a1a2e (어두운 네이비)
- 트랙: #3a3a4a (진한 회색 아스팔트)
- 트랙 경계선: #ffffff 점선
- 시작선: 체크무늬 패턴 (흰+검)
- 커브 연석: 빨간-흰 교차 (curb)
- 잔디(트랙 안쪽): #2a5a2a

### 차량
- 기하학적 도형으로 구성
- 사각형 몸체 (메인 컬러)
- 4개 타이어 (어두운 색)
- 윈드실드 (밝은 색)
- 드리프트 시 타이어 자국 잔상

## 기술 스택
- 엔진: Canvas 2D
- 라우트 그룹: (canvas-mobile)
- 모바일 스케일링: CSS transform
- HUD: 기존 공유 HUD 시스템 사용

## 확장 계획
- K-Racing2, K-Racing3 등으로 다른 트랙 추가 예정
- 현재는 타원형 트랙 1개로 시작
